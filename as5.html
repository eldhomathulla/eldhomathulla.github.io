Conclusion

Load Balancing and Service Discovery using Apache Curator

The implementation of load balancing and service discovery will provide flexibility in terms of the metric for load balancing, but this flexibility comes with extra effort required to implement the load balancing. Along with the fact that the load balancing and service discovery is tightly coupled with the API Server and the client. Considering these facts and performance effect because of two requests need to be sent for a single request, this approach in its current form should not be used for load balancing and service discovery for bringing scalability and fault tolerance to API server.

User Interface Development Approach

Comparing the 2 approaches of single page application and hybrid approach, the hybrid approach provides all the advantages by single page application along with added flexibility of choosing the technologies and framework for development independently. Hence the hybrid approach is the desired way for the user interface development.

Distributed Caching

Performance benchmarks for the django application with memcached and Redis, shows an interesting thing. In this benchmark, the insight that was obtained was that the performance difference for both Redis and Memcached based caching were quite similar for upto 10000 requests. Even so Redis is a better Distributed cache since it has in built fault tolerance and scalability which Memcached lacks, since it relies a lot on the client for the same.
